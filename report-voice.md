# Report voice

A Tufte chart rarely ships alone. It sits inside an HTML report, a slide, or a
recap with a title, a lede, captions, and connective prose. The chart can be
clean and the writing around it can still ruin it. This file governs that prose.

The rule: write like a business, not like a brochure. Plain, rational, named for
the specific case. The reader is busy and literate. Respect that.

## Kill these in any prose the skill emits

| Kill | Why | Replace with |
|---|---|---|
| Em-dashes | Reads as AI cadence | Period, comma, or colon |
| Drama and theatrics ("a staggering leap", "the results are in") | Hype is not information | State the number and what it means |
| Hype verbs (unlock, supercharge, revolutionize, leverage) | Empty | The plain verb (raise, cut, move, change) |
| Vague abstractions ("key learnings", "actionable insights") | Says nothing | Name the actual finding |
| Generic labels ("Metric A", "the data shows") | Not named for this case | The real metric and the real subject |
| Hedge soup ("approximately ~around") | False caution | One number |
| Triplet aphorisms, italic eyebrows, pull-quote flourish | Literary decoration | Plain sentence |

## Keep

- Plain executive register. Short declarative sentences. Vary their length.
- Every label and caption names the specific business case, not a placeholder.
- First the number, then what it means, then what to do. Nothing else.
- One claim per sentence. If a sentence has no number or no decision in it, cut it.

## The test

Read the prose back. If it sounds like a press release, a LinkedIn post, or a
generic AI summary, rewrite it. If a peer would read it and say "obviously, just
tell me the number," you are close.

This mirrors the user's standing writing-style rule: no em-dashes, no AI slop,
rational and named for the specific case.
